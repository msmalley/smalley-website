The Problem

Board game designers need hex maps. Designing them manually is tedious. Dozens or hundreds of hexes, each with terrain assignments, each needing visual consistency. Sharing maps means exporting images or proprietary file formats. Collaborating means sending files back and forth. There was no tool where a map could be a URL.

Architecture

The framework treats hex maps as deterministic functions of their parameters. A seed string plus a configuration (ring count, theme, orientation) always produces the same map. This makes maps shareable as URLs, reproducible across devices, and embeddable without hosting.

  • Seeded RNG — same seed always produces the same terrain distribution. Share a map by sharing its URL
  • Canvas rendering — flat-top or pointy-top orientation, configurable ring count, four distinct visual styles (artistic, classic, kenney, realistic)
  • Theme system — each game has its own terrain palette (volcanic for Nukes, fantasy biomes for Talisman, space for Twilight Imperium, colonial for Colony)
  • Click-to-edit — override any hex's terrain after generation for manual refinement
  • JSON export/import — full board state serialisation for saving and restoring edited maps
  • PNG/PDF export — 2x resolution PNG downloads and A4 page PDF export for print-ready maps
  • SVG export — serialised SVG with annotations (highlights, tokens, arrows, legends) via postMessage API
  • Hex metadata — game-config-driven descriptions with hover info per hex (e.g. all 55 Talisman terrain types with names and action text)

URL Parameter API

Every aspect of the map is controllable via URL parameters, making it trivially embeddable:

  • ?game=nukes — select terrain palette
  • ?size=4 — set board size (ring count)
  • ?seed=volcano — deterministic generation from this seed
  • ?boardonly=1 — minimal chrome for iframe embedding

Use Cases

The framework currently supports four games in the Moddable Games ecosystem: Nukes (area control), Talisman (adventure), Twilight Imperium (4X strategy), and Colony (resource management with 7 Seafarers-style layouts including fog, ports, and gold terrain). Each uses the same hex engine with different terrain definitions, generation rules, and visual themes.

Because maps are URLs, game sessions can be set up by sharing a link. Tournament organisers can publish map seeds in advance. Players can create custom maps and export them as JSON, PNG, PDF, or annotated SVG for inclusion in community map packs or print-ready rulebooks.
